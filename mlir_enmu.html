MLIR Op 设计革新

探索一种更简洁、高效的 `EnumCreateOp` 实现方式：
将枚举变体信息作为 Attribute 传递

核心思想：化繁为简

传统方法：SSA Operands

每个枚举变体都作为独立的SSA操作数传入。这导致IR冗长，定义-使用链复杂，后端处理分散。

// 概念性IR，非真实
%c0 = arith.constant 0 : i32
%c1 = arith.constant 1 : i32
%c2 = arith.constant 2 : i32

%enum = pdlir.enum.create "Red", %c0, "Green", %c1, "Blue", %c2
🔗 IR冗余，处理复杂

新设计：Attributes

所有变体信息被打包成一个结构化的`ArrayAttr`属性。IR更简洁，信息集中，后端处理更直接。

// 推荐的IR设计
%enum = pdlir.enum.create "Color" {
variants = [
{ name = "Red", value = 0 : i32 },
{ name = "Green", value = 1 : i32 },
{ name = "Blue", value = 2 : i32 }
]
}
📦 IR简洁，信息集中

技术实现深潜

使用TableGen声明式地定义`EnumCreateOp`。关键在于将`variants`定义为一个`ArrayAttr`类型的属性，而不是可变数量的`arguments`。

def pdl_EnumCreateOp : pdl_Op<"enum.create"> { let summary = "创建枚举类型并定义其变体。"; let arguments = (ins); // 没有SSA操作数 let attributes = [ StringAttr:$enumName, ArrayAttr:$variants // 所有变体信息在此 ]; let results = (outs Type:$enumType); let assemblyFormat = "$enumName attr-dict `:` type($enumType)"; let hasVerifier = 1; }

在Python代码生成器中，收集所有变体信息，构造一个`DictAttr`的列表，最后用`ArrayAttr.get()`将其包装起来，传递给`EnumCreateOp`的构造函数。

def visitEnum(enum_ty, enum_name, fields): variant_attrs = [] for name, value in fields: variant_dict = DictAttr.get({ "name": StringAttr.get(name), "value": IntegerAttr.get(i32_type, value) }) variant_attrs.append(variant_dict) variants_attr = ArrayAttr.get(variant_attrs) enum_name_attr = StringAttr.get(enum_name) return pdlir.EnumCreateOp( enum_name_attr, variants_attr, enum_ty )

C++后端遍可以直接从操作中获取`variants`属性，然后遍历这个数组来访问每一个变体的名称和值，处理逻辑非常直接。

void processEnum(pdlir::EnumCreateOp op) { llvm::outs() << "Enum: " << op.getEnumName() << "\n"; mlir::ArrayAttr variants = op.getVariants(); for (mlir::Attribute attr : variants) { auto dictAttr = attr.cast<mlir::DictionaryAttr>(); auto name = dictAttr.get("name").cast<mlir::StringAttr>(); auto value = dictAttr.get("value").cast<mlir::IntegerAttr>(); llvm::outs() << " - " << name.getValue() << " = " << value.getInt() << "\n"; } }

新设计带来的优势

✨

IR 更简洁

消除了为每个变体生成`arith.constant`的冗余操作，使IR更加紧凑和可读。

⚙️

后端更易处理

后端只需解析单个属性即可获得所有枚举项，无需遍历复杂的SSA定义-使用链。

🚀

易于扩展

未来可轻松在`DictAttr`中添加文档、元数据等新字段，而不破坏现有结构。
